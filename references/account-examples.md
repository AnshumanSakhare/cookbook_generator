# Account Examples

Use these to match the account's natural rhythm, not to copy line-for-line.

## Repeating Patterns

- lead with the product name and the concrete change
- compress multiple specs into one practical takeaway
- use a final line that reframes why builders should care
- prefer "this replaces...", "which means...", "worth paying attention to...", "what's worth noting..."
- use bullets only when they increase scan speed

## Strong Native Examples

### Simple product reaction

`Gemini Embedding 2 just dropped.`

`one embedding model for`
`text + images + audio + video + docs`
`no separate pipelines for each modality anymore.`

`which means retrieval across formats becomes much simpler.`

Why it works:
- immediate hook
- concrete change
- direct implication for builders

### Focused model take

`Qwen Image 2 model is worth paying attention to if you work with text-heavy visuals.`

`it has dedicated typography engine - handles dense layouts, multi-script, slide-ready content`

`native 2K output, 7B params, faster inference than v1`

`text rendering in image models has always been the weak spot and this one takes a different approach to it.`

Why it works:
- specific audience
- specs in context
- grounded take instead of hype

### Infrastructure / workflow angle

`Perplexity Computer now has skills.`

`Define a workflow once.`
`Computer runs it automatically when needed.`

`Support triage, compliance checks, marketing analysis, all reusable.`

`This replaces a lot of small manual steps builders keep repeating.`

Why it works:
- short lines
- clear workflow framing
- direct "stack replacement" angle

### Grounded feature explanation

`Indian Startup @SarvamAI ships something most models don't include by default.`

`its "wikipedia grounding"`

`It pulls facts from Wikipedia before answering`
`instead of guessing from its training data.`
`Fewer hallucinations on factual questions.`

Why it works:
- identifies the unusual feature
- explains it in plain language
- makes the benefit obvious

## Posts That Reveal House Style

- `Google open-sourced the Agent Development Kit`
- `always-on memory agent pairs with Gemini 3.1 Flash-Lite, runs 24/7 as a background process, no vector DB needed.`
- `100% open source. the "agents forget everything" problem has an actual solution now.`

- `GPT-5.4 just dropped in the API.`
- `reasoning + coding + agents + native computer use. 1M token context on top of that.`
- `built a cookbook to get you started`

- `Sarvam just open-sourced 30B and 105B models.`
- `built from scratch - data, research, inference optimization, all in-house.`
- `weights are on Hugging Face right now.`

## Anti-Patterns Seen In Older Posts

Avoid drifting into these:
- broad bragging without a builder implication
- emoji-heavy hype
- generic benchmark recap
- "AI king" style phrasing
- repeating official release language without adding a take

If a draft sounds more like these anti-patterns than the stronger examples above, rewrite it.
